# 文档审核助手

这是一个面向古典文献教学场景的前端工具，主要用于上传教学标准、审核学生文稿、进行学术分析、管理知识库，以及进行经典阅读与批注。

项目目前是一个不依赖构建工具的静态页面应用，直接由 `index.html` 作为入口运行，适合快速演示、课程内部使用和后续继续拆分迭代。

## 功能概览

### 1. 教学文档管理

- 支持上传多个教学文档作为审核标准。
- 支持直接粘贴教学内容。
- 支持将教学标准保存到本地知识库。
- 支持把教学标准上传到社区供他人复用。

### 2. 文档审核

- 支持上传待审核文档并进行 AI 审核。
- 支持对同名文档进行版本迭代记录与对比。
- 支持展示评分结果、审核意见和进步情况。

### 3. 深度学术分析

- 核心论点质疑
- 薄弱环节检测
- 学术立场分析
- 内证互参
- 训诂释义
- 跨文献关联

### 4. 经典阅读与批注

- 提供经典文本阅读界面。
- 支持逐句选中、自动检测和本地批注。
- 支持全屏阅读模式。

### 5. 知识库与社区

- 本地知识库存储教学文档和审核文档。
- 社区模块支持文档分享、评论、全屏阅读和批注。

## 使用方式

1. 直接打开 `index.html`。
2. 上传教学文档，或直接粘贴教学标准内容。
3. 点击“理解文档”，生成教学标准摘要。
4. 上传需要审核的文档。
5. 查看审核结果、评分和版本变化。
6. 根据需要使用右侧的深度分析能力。
7. 进入知识库或社区继续管理和复用文档。

## 支持格式

- `.txt`
- `.md`
- `.docx`
- `.pdf`
- 常见图片格式，用于 OCR 识别

## 数据与存储说明

- 主要业务数据默认保存在浏览器本地存储中。
- 社区功能使用 Supabase 作为远端数据服务。
- `references.js` 提供项目运行所需的内置经典文献参考数据。

## 文件结构说明

下面这部分专门说明每个文件和目录的内容与作用，方便后续维护、查问题和继续拆分。

### 入口与基础数据文件

| 文件 | 作用 |
|------|------|
| `index.html` | 项目的页面入口，负责定义整体 DOM 结构，并按顺序引入外部依赖、样式文件和脚本文件。所有页面板块的骨架都在这里。 |
| `references.js` | 内置学术参考数据文件，提供经典文献索引、训诂释义、跨文献关联等数据，供审核和分析模块调用。 |
| `README.md` | 项目说明文档，用于介绍功能、结构、使用方式以及各文件职责。 |

### `scripts/` 目录说明

| 文件 | 作用 |
|------|------|
| `scripts/core.js` | 核心公共模块，负责全局配置、基础状态、标签页切换、通用初始化、公共 API 调用和基础交互逻辑。 |
| `scripts/file-processing.js` | 文件处理模块，负责上传文件后的解析流程，包括文本读取、`docx` 解析、PDF 提取、OCR 识别、格式信息提取和预览切换。 |
| `scripts/knowledge-base.js` | 知识库模块，负责本地知识库的读取、保存、清理、导出、搜索，以及内置参考数据与自定义数据的合并。 |
| `scripts/teaching.js` | 教学文档模块，负责教学标准的上传、列表渲染、删除、摘要生成、保存到知识库和上传到社区。 |
| `scripts/review.js` | 文档审核模块，负责待审核文档的提交、评分、版本对比、审核报告生成等主流程。 |
| `scripts/analysis.js` | 深度分析模块，负责右侧分析面板相关能力，包括引文分析、训诂分析、跨文献分析，以及几种 AI 深度分析功能。 |
| `scripts/community.js` | 社区模块，负责 Supabase 连接、社区文档加载、评论管理、上传分享、全屏阅读和阅读批注。 |
| `scripts/reference-detection.js` | 参考文献自动识别模块，负责识别文本中的经典引用、推断章节、查询本地参考数据，并尝试做外部补充。 |
| `scripts/classic-reading.js` | 经典阅读模块，负责经典文本展示、句子切分、选句高亮、自动检测、批注保存和全屏阅读。 |

### `styles/` 目录说明

| 文件 | 作用 |
|------|------|
| `styles/base.css` | 基础样式文件，包含全局重置、主题变量、字体和页面基础外观设置。 |
| `styles/layout.css` | 布局样式文件，负责头部、主内容区、左右侧边栏等整体布局。 |
| `styles/panels.css` | 面板样式文件，负责教学文档、审核文档、知识库、社区等主内容区域的卡片、按钮、表单和结果展示。 |
| `styles/reader.css` | 阅读相关样式文件，负责右侧分析栏、全屏阅读器、批注区、弹窗等阅读交互界面。 |
| `styles/responsive.css` | 响应式样式文件，负责平板和移动端下的布局压缩与适配。 |

## 模块关系说明

- `index.html` 是页面结构层。
- `styles/` 是页面样式层。
- `scripts/` 是页面逻辑层。
- `references.js` 是分析功能依赖的数据层。

当前拆分方式遵循的是“按职责拆分”而不是“按页面切分”：

- 公共能力放在 `core.js`
- 文件解析放在 `file-processing.js`
- 业务能力按教学、审核、分析、社区、阅读分别拆开
- 样式按基础、布局、面板、阅读器、响应式拆开

这样的好处是后续继续维护时更容易定位问题，也更适合继续往组件化或工程化方向演进。

## 脚本加载顺序说明

当前 `index.html` 中的脚本是按依赖顺序加载的，维护时尽量不要随意调整：

`core.js` -> `file-processing.js` -> `knowledge-base.js` -> `teaching.js` -> `review.js` -> `analysis.js` -> `community.js` -> `reference-detection.js` -> `classic-reading.js`

原因是后面的模块会直接依赖前面模块提供的公共方法、配置或状态。

## 后续维护建议

- 如果新增功能，优先放进已有职责最接近的脚本文件中。
- 如果某个功能继续膨胀，建议再向下拆分子模块，而不是重新塞回 `index.html`。
- 如果未来引入构建工具，可以优先把 `scripts/` 进一步改造成更明确的模块依赖关系。
